## Migration Plan: Consolidate to brunogarcia.com

### Current State
- **brunogarcia.com** — static landing page with social icons, hosted on Firebase
- **garcia.in** — Hugo blog (PaperMod), 2 posts + 9 migrated from Blogger, hosted on GitHub Pages
- **blog.brunogarcia.com** — old Blogger blog, 25 posts, ~13 years of domain history, still live

### Target State
- **brunogarcia.com** — Hugo blog (everything), hosted on Firebase (or GitHub Pages)
- **garcia.in** → 301 redirect to brunogarcia.com
- **blog.brunogarcia.com** → 301 redirects to brunogarcia.com (per-post where migrated, homepage for rest)

---

### Phase 1: Merge blog into brunogarcia.com repo
- [ ] Change `baseURL` in `hugo.yaml` from `garcia.in` to `brunogarcia.com`
- [ ] Update CNAME/DNS accordingly
- [ ] Move Hugo site to brunogarcia.com repo (or just repoint Firebase)
- [ ] Update `canonical` URLs everywhere
- [ ] Keep Firebase hosting for brunogarcia.com OR switch to GitHub Pages
  - Firebase: already has domain, just change `site/` to Hugo's `public/` output
  - GitHub Pages: move domain from Firebase, simpler CI (already have workflow)

### Hosting Decision
**Recommended: GitHub Pages**
- Already have CI workflow
- Free HTTPS
- No Firebase dependency
- Move brunogarcia.com DNS to GitHub Pages

**Alternative: Firebase**
- Already configured for brunogarcia.com
- Would need to update deploy workflow to `firebase deploy` instead of GitHub Pages
- Supports proper 301 redirects via `firebase.json`

- Phase 5: After confirming Google has re-indexed (2-4 weeks)
